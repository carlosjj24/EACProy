Git Flow - Lección 2

Master y Develop son intocables

       Todo lo que nosotros hagamos como desarrolladores terminará, tarde o temprano, en alguna de estas ramas.

       Develop: Rama que contiene las features a incluir en un próximo deploy a producción. Todos los nuevos módulos se guardarán en Develop, a la espera de salir a producción.

       Master: Rama que contiene el código que está en producción. A esta rama solo debe llegar código que está listo para entrar en producción.

       ¡Oh! Pero hay veces en donde me gustaría hacer una pequeña modificación directamente sin crear ramas extra. ¡NO! Ya veremos por qué.



Elementos básicos - Feature

       No es otra cosa que una rama de Git con una copia exacta del contenido de Develop.

       Pero si habíamos dicho que Develop no se tocaba... Exactamente, para poder realizar cambios sobre el código, generamos una copia exacta del contenido de Develop y lo metemos en una nueva rama llamada feature.

       El propósito de las features es permitir el desarrollo de nuevas funcionalidades del proyecto, en un ambiente controlado, de forma tal de que, una vez listos, se agreguen (merge) a Develop.

       La razón de hacer esto es muy fácil de argumentar: si todos trabajan sobre la misma rama, podrían generarse muchos conflictos.



Elementos básicos - Feature

       Los nombres de todas las features comienzan con la palabra feature, seguido de una barra invertida y la referencia para identificarla.

       Por ejemplo:

       feature/ADD_FACEBOOK_AUTHENTICATION

       feature/RemoveCvvFromCreditCardModel



Elementos básicos - Feature



Elementos básicos - Hotfix

       Un hotfix es una rama con una copia exacta del contenido de Master.

       Imagina que la versión que está ejecutándose en producción tiene algún bug que debe ser corregido de forma inmediata. Esa versión es idéntica a lo que está en la rama Master, por lo que estaría bien trabajar sobre el código de dicha rama.

       Cuando estemos seguros de haber corregido el bug, podemos cerrar el hotfix para que se agregue a Master.

       Pero no basta solo con agregarlo a Master, es necesario agregarlo a Develop también. ¡Ojo!

       Se sigue manteniendo la misma forma de nombrado como en los features. Por ejemplo: hotfix/1.10.1



Elementos básicos - Hotfix



Elementos básicos - Release

       Un release es una rama con la copia exacta del contenido de Develop, que está lista (o casi) para ser subida a producción.

       En otras palabras, es el intermediario entre Develop y Master.

      Si ocurre un bug, este será resuelto en la misma rama Release. Así el bug también será corregido en la rama Develop que nos trajimos.
