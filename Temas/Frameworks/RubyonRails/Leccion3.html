Curiosidades y Hola Mundo - Lección 3

¿Cual es el proposito de un controlador?

       El propósito de un controlador es recibir las peticiones (requests) de la aplicación. El enrutamiento (routing) decide qué controlador recibe qué petición. A menudo, hay más de una ruta para cada controlador, y diferentes rutas pueden ser servidas por diferentes acciones (actions). El propósito de cada acción es obtener información para pasarla después a la vista.

       Para crear un nuevo controlador, necesitas ejecutar el generador de controladores y decirle que quieres un controlador llamado por ejemplo welcome con una acción llamada index. Para ello, ejecuta lo siguiente:

            • $ rails generate controller welcome index



¿Cual es el proposito de un controlador?

Rails creará una serie de archivos y añadirá una ruta por ti.

       Los archivos más importantes de éstos son por supuesto el controlador, que se encuentra en app/controllers/welcome_controller.rb y la vista, que se encuentra en app/views/welcome/index.html.erb. Abre el archivo app/views/welcome/index.html.erb en tu editor de texto y edítalo para que contenga sólo está línea de código:

            • ¡Hola mundo!



¿Cual es el proposito de una vista?

El propósito de una vista es mostrar la información en un formato legible para los humanos. Una distinción importante que hacer es que es el controlador, y no la vista, donde la información es recolectada. La vista sólo debería mostrar la información. Por defecto, las plantillas de las vistas están escritas en un lenguaje llamado ERB (del inglés, Embedded Ruby), que se procesa automáticamente para cada petición servida por Rails.



¿Que es Rails?

    Rails es un framework para el desarrollo de aplicaciones web, software libre por naturaleza, está basado en el patrón de diseño Modelo Vista Controlador (MVC). Fue creado por David Heinemeier Hansson, empleado de la empresa 37signals.

    Fue liberado por primera vez al público en julio del 2004, y lo implemento en una aplicación orientada a la administración de proyectos llamada Basecamp. Rails está basado en estos principios de desarrollo:

  • Don’t Repeat Yourself

  • Convention Over Configuration



Primer principio: Don’t Repeat Yourself

    No lo vuelvas a repetir”, es una de las cosas más novedosas funcionalidades de este framework. Imagina tener un formulario, y llamarlo las veces que se desee y desde cualquier lugar del codigo, simplemente con una línea código, pues ese deseo lo vuelve real Rails.

    Tener una tabla en tu base de datos, y manipular a los registros como un objeto y a sus campos como un atributo, sin necesidad de declarar nada, son sólo algunas aplicaciones de este principio de desarrollo.



Segundo Principio: Convention Over Configuration

    “Convención antes que Configuración”, Rails trae todo configurado por defecto ayudando al programador a ahorrar tiempo, si el programador así lo desea puede desconfigurar o cambiar la configuración, pero en principio si lo necesita ya estará configurado.



¿Qué es Ruby on Rails?

    Ruby on Rails es un entorno de desarrollo web de código abierto que está optimizado para la satisfacción de los programadores y para la productividad sostenible. Te permite escribir un buen código evitando que te repitas y favoreciendo la convención antes que la configuración.
