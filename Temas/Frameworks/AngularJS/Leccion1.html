Conceptos Basicos de AngularJS - Lección 1

¿Qué es AngularJS?

       AngularJS es Javascript. Es un proyecto de código abierto, realizado en Javascript que contiene un conjunto de librerías útiles para el desarrollo de aplicaciones web y propone una serie de patrones de diseño para llevarlas a cabo. En pocas palabras, es lo que se conoce como un framework para el desarrollo, en esta caso sobre el lenguaje Javascript con programación del lado del cliente.



Características de AngularJS

Entre las características de este framwork se encuentran:

  • Modularidad, es decir podemos utilizar componentes con módulos, esto trae como consecuencia una inyección de dependencia.

  • Existen numerosas ventajas listas para usar en los enrutadores, validación de formularios, HttpClient, etc.

  • El alto nivel de control, a diferencia de React, Angular permite el uso normal de HTML y CSS junto con todas sus características y ventajas.

  • Apoyo total de una de las grandes empresas en TI (Google)

  • Angular está basado en JavaScript, algunos desarrolladores lo ven como una gran ventaja.



¿En que patron de diseño se basa?

    AngularJS se basa en el patron de diseño MVC (Modelo-Vista-Controlador), donde cada uno de estos tiene asociadas diferentes funciones.

    ¿Como se comporta el Modelo-Vista-Controlador en AngularJS?

  • Vistas: Será el HTML y todo lo que represente datos o información

  • Controladores: Se encargarán de la lógica de la aplicación y sobre todo de las llamadas "Factorías" y "Servicios" para mover datos contra servidores o memoria local en HTML5.

  • Modelo de la vista: En Angular el "Modelo" es algo más de aquello que se entiende habitualmente cuando te hablan del MVC tradicional, osea, las vistas son algo más que el modelo de datos. En modo de ejemplo, en aplicaciones de negocio donde tienes que manejar la contabilidad de una empresa, el modelo serían los movimientos contables. Pero en una pantalla concreta de tu aplicación es posible que tengas que ver otras cosas, además del movimiento contable, como el nombre de los usuarios, los permisos que tienen, si pueden ver los datos, editarlos, etc. Toda esa información, que es útil para el programador pero que no forma parte del modelo del negocio, es a lo que llamamos el "Scope" que es el modelo en Angular.

    Nota: Por ese motivo por el cual en AngularJS tienes unos modelos un poco diferentes, algunos autores dicen que el patrón que utiliza el framework es el MVVM (Model-View-View-Model).



Velocidad del renderizado

    El renderizado en una app con angular está estrechamente relacionado con el NgModelController, el cual es un controlador que proporciona funciones para la directiva ng-model, que es la que se encarga de permitir que los datos viajen entre modelo <=> vista (comunicación bidireccional). El controlador contiene servicios para enlace de datos, validación, actualizaciones de propiedades CSS y análisis de valores, sin embargo este no contiene ninguna lógica que se ocupe de la representación de DOM o de escuchar eventos, dicha lógica debe ser proporcionada por otras directivas de vinculación de datos para controlar elementos que hagan uso del NgModelController.



Estructura de ficheros

    Con esta estructura de fichero propuesta por la comunidad que respalda AngularJS, se plantea con la finalidad de facilitarle al equipo de desarrolladores de una aplicación una estructura que garantice una escalabilidad ordenada y mantenible en el tiempo, ya que modulariza los componentes que se usan dentro de la aplicacion haciendolo mas flexible y receptivo a la hora de aplicar cambios o añadir nuevas funcionalidades.
