Funciones de un API REST - Lección 2

¿Qué permite un API REST?

       Las API REST proporcionar interoperabilidad como una característica básica, es decir, que todas las aplicaciones puedan comunicarse con otras aplicaciones sin depender del sistema operativo subyacente y los lenguajes de programación, esta interoperabilidad se permite mediante el uso de hipermedios para permitir al usuario navegar por los distintos recursos de una API REST a través de enlaces HTML.

      Dato Curioso: hipermedio es el término usado en el ámbito de las páginas web para definir el conjunto de procedimientos para crear contenidos que contengan texto, imagen, vídeo, audio y otros métodos de información.



¿Qué es API REST?

Formatos populares de solicitud de API REST:

  • REST

  • XML-RPC

  • SOAP

Formatos populares de respuesta de API REST:

  • REST

  • XML-RPC

  • SOAP

  • JSON

  • PHP



Proceso de un API REST



Como funciona REST

Consultas o llamadas al API

    Las llamadas al API se implementan como peticiones HTTP, en las que:

  • La URL representa el recurso. Ejemplo: Ejm: API de Github, https://api.github.com/users/ + {{username}}

  • El método GET es el método HTTP que se usa para invocar esta operación. Ejemplo: $http.get("https://api.github.com/users/" + username)

    Como respuesta inicial se recibe un código de estado HTTP, el cual representa el resultado, un codigo 200 representa un estado OK HTTP/1.1, es decir devuelve la respuesta que el desarrollador esperaba, por el contrario el código 404 representa un NOT FOUND HTTP/1.1, el cual representa que la petición no fue exitosa.



Como funciona REST

Creacion de recursos o modificación de recursos

    El método HTTP que se usa para la creación de elementos dentro de un API ya existente es el método POST, pasándole como parámetros la información que será añadida al recurso.

    Respuesta a la creación de recursos

  • 403, Acceso prohibido.

  • 400, petición incorrecta, p.ej. falta un campo o su valor no es válido.

  • 500, Error del lado del servidor al intentar crear el recurso, p.ej. se ha caído la BD.

  • 201, Recurso creado correctamente.

¿Qué URL tiene el recurso recién creado?

      La convención en REST es devolverla en la respuesta como valor de la cabecera HTTP Location.

Como funciona REST

Actualización de recursos

    El método HTTP que se usa para la creación de elementos dentro de un API ya existente es el método POST, pasándole como parámetros la información que será añadida al recurso.

  Metodo PUT

  • Según la ortodoxia REST, actualizar significaría cambiar TODOS los datos.

  • PATCH es un nuevo método estándar HTTP (2010) pensado para cambiar solo ciertos datos. Muchos frameworks de programación REST todavía no lo soportan.

  Resultados posibles

  • Errores ya vistos con POST.

  • 201, Recurso creado, cuando le pasamos el id deseado al servidor.

  • 200, Recurso modificado correctamente.

Como funciona REST

Eliminar recursos

    Metodo DELETE.

      Algunos resultados posibles:

  • 200, OK, recurso eliminado correctamente

  • 404 Not found.

  • 500 Server error.

  NOTA: Tras ejecutar el DELETE con éxito, las siguientes peticiones GET a la URL del recurso deberían devolver 404.
