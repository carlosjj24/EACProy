Conceptos Basicos de API REST - Lección 1

¿Qué es un API?, ¿Qué es REST?

       API: Es un conjunto de reglas y especificaciones que las aplicaciones pueden seguir para comunicarse entre ellas. Para que lo entendamos, el uso de una API es el mecanismo más útil para conectar dos softwares entre sí.

      REST: El término REST (Representational State Transfer) se originó en el año 2000, descrito en la tesis de Roy Fielding, padre de la especificación HTTP. Un servicio REST no es una arquitectura software, sino un conjunto de restricciones con las que podemos crear un estilo de arquitectura software, la cual sonun conjunto de funciones que los programadores pueden usar para enviar solicitudes y recibir respuestas utilizando los métodos de protocolo HTTP como GET y POST.



¿Qué es API REST?

       La API REST puede ser utilizada por cualquier sitio o aplicación sin importar en qué idioma esté escrito porque las solicitudes se basan en el protocolo HTTP universal, y la información generalmente se devuelve en formato JSON que casi todos los lenguajes de programación pueden leer.

Dato Curioso: El termino RESTful, es comúnmente utilizado para referirse a las aplicaciones que implementan servicios REST.



Restricciones que definen un sistema RESTful

    Cliente-servidor: esta restricción mantiene al cliente y al servidor débilmente acoplados. Esto quiere decir que el cliente no necesita conocer los detalles de implementación del servidor y el servidor se “despreocupa” de cómo son usados los datos que envía al cliente.

    Sin estado: aquí decimos que cada petición que recibe el servidor debería ser independiente, es decir, no es necesario mantener sesiones.

    Sistema de capas: el servidor puede disponer de varias capas para su implementación. Esto ayuda a mejorar la escalabilidad, el rendimiento y la seguridad.



Restricciones que definen un sistema RESTful

        Interfaz uniforme: define una interfaz genérica para administrar cada interacción que se produzca entre el cliente y el servidor de manera uniforme, lo cual simplifica y separa la arquitectura. Esta restricción indica que cada recurso del servicio REST debe tener una única dirección, “URI”

        Cacheable: debe admitir un sistema de almacenamiento en caché. La infraestructura de red debe soportar una caché de varios niveles. Este almacenamiento evitará repetir varias conexiones entre el servidor y el cliente para recuperar un mismo recurso.
