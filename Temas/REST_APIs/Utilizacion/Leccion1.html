Caso de Estudio API REST de GitHub - Lección 1

API REST de GitHub

       En este caso de estudio aplicaremos en principio un estudio de API de GitHub, para usarlos en una serie de vistas. Con la finalidad de buscar por el nombre de usuario en GitHub haciendo conexión con su API y ver todos los repositorios que tiene agregado el usuario a su canal, detallando por cada repositorio del usuario las estrellas, el lenguaje bajo el cual esta escrito y el link a dicho repositorio.

      NOTA: Para la implementacion de esta API REST utilizaremos el appoyo del framework AngularJS.



¿Qué se desea obtener?

Un buscador de usuarios para detallar su informacion.

Detalles del usuario buscado: el Avatar del usuario, el nombre y una tabla un donde se evidencie por repositorio del usuario: el nombre, la cantidad de estrellas, el lenguaje, URL. Además que se pueda ordenar dicha tabla por cualquiera de los tres primeros campos (Nombre, Estrellas, Lenguaje)


Estructura del API REST de GitHub

    Peticion o solicitud: como toda API REST el consumo de los datos se realiza por el metodo GET.

    Respuesta: y datos que nos interesan de la misma .



Implementación

En primera instacia se deben definir las vistas o pantallas de la aplicacion.

     Ng-repeat hace referencia a la directiva que aplica una función parecida al forEach de Js, en este caso recorre el arreglo de repositorios y por posición en el arreglo extrae la información requerida “name” = nombre del repositorio, “stargazers_count” = la cantidad de estrellas por repositorio y convierte dicho dato en numero o entero, el “language” = al lenguaje del repositorio y por ultimo “html_url” = url del repositorio en GitHub.
